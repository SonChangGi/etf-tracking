Methodology

계산 방법과 한계

가격 효과

전일 전체 보유종목 비중에 개별 종목의 KRW 평가단가 수익률을 적용하고, 전체 보유종목 가중 평균 수익률로 정규화한 no-trade 예측 비중을 계산합니다.

잔차 신호

실제 비중 변화가 예측 비중 변화보다 작지만 방향성이 있으면 약한 매수/매도 관찰 설명을 붙이고, 허용오차 이상 크면 likely_buy 또는 likely_sell로 표시합니다. 모두 실제 주문 확정이 아닌 no-trade 기준 대비 잔차입니다.

커버리지

수익률이 확보된 전일 전체 보유종목 비중 합계가 60% 미만이면 가격/매매 요인을 단정하지 않고 mixed 또는 낮은 신뢰도로 표시합니다.

히스토리 범위

검토된 수동 실행은 최신 데이터를 먼저 가져온 뒤 최근 구간을 갱신합니다. 더 과거 분석은 수동 백필로 확장할 수 있으며, 정적 JSON 용량과 공급자 throttling을 함께 고려합니다.

Manual Update

없는 날짜만 채우는 수동 업데이트

수동 워크플로는 이미 저장된 usable 스냅샷을 다시 요청하지 않고, 최신 기준일과 선택한 백필 범위 안에서 아직 없는 날짜만 채웁니다. 공급자 throttling을 줄이면서 기존 히스토리는 보존합니다.

보안상 공개 정적 페이지에는 GitHub 토큰을 저장하지 않습니다. 버튼은 인증된 GitHub Actions 실행 화면으로 연결하며, 저장소 쓰기 권한이 있는 계정으로 로그인한 뒤 Run workflow를 누르면 됩니다.

  • 자동 스케줄 중지멀티 repo 롤백 이후 예약 실행은 비활성화되어 있습니다.
  • 검토 후 수동 실행공급자/전일 종가 지연을 확인한 뒤 workflow_dispatch로 실행합니다.
  • 수동 실행backfill_start_date로 과거 누락 구간을 채우고, 필요할 때만 refresh_existing을 사용합니다.
GitHub Actions에서 수동 업데이트 열기 워크플로 상태 보기 gh workflow run update-data.yml --repo SonChangGi/etf-tracking --ref main -f backfill_all=false -f backfill_start_date= -f refresh_existing=false -f strict_validation=false 기본값은 missing-only입니다. 실행 후 Actions 상태와 대시보드 생성 시각을 확인하세요.
